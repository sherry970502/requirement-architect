# 核心价值结构化（Core Value Structure）

这是一套将核心价值系统化分析和表达的方法。核心逻辑是：**识别所有干系人，针对每个干系人分析他们从产品中获得的最核心价值，并以标准句式清晰表达，最后按重要程度排序**。

分析分为 4 个步骤，最终输出一份按优先级排序的核心价值清单。

---

## 核心概念定义

### 什么是核心价值？

**核心价值**是指产品为干系人提供的价值中最核心、最关键的部分。它描述的是干系人从产品的功能、内容或服务中获得的最主要好处，是干系人选择产品的重要原因。

### 核心价值的标准句式

```
为谁（干系人）+ 实现什么价值 + 通过什么方式
```

**示例：**
- ✅ 正确：「为在线学习的学生，实现个性化学习路径规划，通过 AI 分析学习数据自动推荐课程」
- ✅ 正确：「为公司，实现品牌知名度提升，通过虚拟 IP 在社交媒体的持续曝光」
- ❌ 错误：「提升用户体验」（没有明确干系人、价值不够具体、缺少方式）
- ❌ 错误：「让用户更方便」（价值模糊、缺少方式）

---

## 三个核心要素

### 1. 干系人（Who）

**内容：** 核心价值的受益者，可以是某一类群体、组织或个人。

**目的：** 明确核心价值是为谁服务的，以便提炼的价值更加准确、有针对性。

**识别标准：**
- 谁会直接或间接受益于这个产品？
- 谁的决策会影响产品的成败？
- 谁会为产品付费或提供资源？

**常见干系人类型：**
- 直接用户（使用产品的人）
- 间接用户（受产品影响但不直接使用的人）
- 公司/组织（产品的拥有者或运营者）
- 合作方/供应商
- 投资人/股东
- 其他利益相关方

---

### 2. 实现什么价值（What Value）

**内容：** 描述干系人从产品的功能、内容或服务中获得的最主要好处。

**目的：** 确保价值的表述更客观、理性、准确。

**表述要求：**
- 清晰明确（避免模糊词汇如"提升""优化""改善"）
- 聚焦核心（只描述最主要的好处，不是所有好处）
- 可理解（用干系人的语言表达，不用专业黑话）

**价值类型参考：**
- 效率提升（节省时间、降低成本、提高产出）
- 体验改善（更便捷、更愉悦、更有趣）
- 能力增强（学到新技能、获得新能力）
- 问题解决（解决痛点、消除障碍）
- 情感满足（获得认同、建立连接、感到安全）
- 商业收益（增加收入、扩大市场、提升品牌）

---

### 3. 通过什么方式（How）

**内容：** 描述价值实现的方式或手段。

**目的：** 为核心价值的高质量达成提供方法指引。

**表述要求：**
- 具体可行（不是空泛的"提供服务"，而是具体的实现方式）
- 体现产品特色（说明这个产品用什么独特方式实现价值）
- 简洁明了（不需要详细的技术细节，点明核心方式即可）

**方式类型参考：**
- 技术手段（如 AI 推荐、3D 渲染、实时同步）
- 功能特性（如 24 小时直播、一键生成、智能匹配）
- 内容服务（如专家课程、定制化内容、社区互动）
- 流程优化（如简化步骤、自动化处理、智能引导）

## 分析流程

## Step 1：识别所有干系人

基于原始需求，识别出所有可能从产品中受益的干系人。

**识别方法：**
1. 列出产品的直接用户
2. 思考谁会间接受益
3. 考虑谁会为产品买单或提供支持
4. 识别谁的决策会影响产品成败

**输出：**

```
干系人识别：

干系人 1：[干系人名称]
  类型：[直接用户/间接用户/公司/合作方等]
  重要程度：[对项目的重要性说明]

干系人 2：[干系人名称]
  类型：...
  重要程度：...

（列出所有干系人，通常 3-6 个）
```

---

## Step 2：针对每个干系人分析核心价值

逐一分析每个干系人从产品中获得的核心价值。

**分析框架：**

```
干系人：[干系人名称]

【实现什么价值】
- 核心价值 1：[描述主要好处]
  价值类型：[效率提升/体验改善/能力增强等]
  
- 核心价值 2：[描述次要好处]（如有）
  价值类型：...

【通过什么方式】
- 方式 1：[具体实现方式]
- 方式 2：[具体实现方式]（如有）

【初步整合】
完整表述：为 [干系人]，实现 [价值 1]，通过 [方式 1]
```

**分析要求：**
- 每个干系人至少识别 1 个核心价值
- 如果有多个核心价值，按重要性排序
- 方式要与价值匹配（不能张冠李戴）

**输出：**

针对每个干系人，输出上述分析框架的完整内容。

---

## Step 3：整合成标准句式

将每个干系人的核心价值整合成标准句式。

**整合原则：**
- 一个干系人可能有多条核心价值（如果有多个不同的价值维度）
- 每条核心价值都要符合标准句式
- 确保表述清晰、准确、完整

**输出：**

```
核心价值整合：

干系人 1：[干系人名称]
  核心价值 1：为 [干系人]，实现 [价值]，通过 [方式]
  核心价值 2：为 [干系人]，实现 [价值]，通过 [方式]（如有）

干系人 2：[干系人名称]
  核心价值 1：...
  核心价值 2：...

（所有干系人的核心价值完整列表）
```

---

## Step 4：优先级排序

按照干系人的重要程度和对项目的重要程度，对所有核心价值进行排序。

**排序标准：**

1. **干系人的重要程度**：
   - 核心干系人（产品成败的关键）> 重要干系人 > 次要干系人
   - 直接用户通常比间接用户重要
   - 付费方通常比非付费方重要

2. **对项目的重要程度**：
   - 这个价值是否是项目存在的根本原因？
   - 这个价值是否影响项目的核心竞争力？
   - 这个价值是否直接影响商业目标达成？

**排序方法：**
- P0（核心必备）：最核心的干系人 + 最关键的价值
- P1（重要支撑）：重要干系人 + 重要价值
- P2（次要补充）：次要干系人 + 补充价值

**输出：**

```markdown
# 核心价值清单（按优先级排序）

## P0 级 - 核心必备

| 序号 | 核心价值 | 干系人 | 优先级 |
|------|---------|--------|--------|
| 1 | 为 [干系人]，实现 [价值]，通过 [方式] | [干系人名称] | P0 |
| 2 | 为 [干系人]，实现 [价值]，通过 [方式] | [干系人名称] | P0 |
| ... | ... | ... | ... |

## P1 级 - 重要支撑

| 序号 | 核心价值 | 干系人 | 优先级 |
|------|---------|--------|--------|
| X | 为 [干系人]，实现 [价值]，通过 [方式] | [干系人名称] | P1 |
| ... | ... | ... | ... |

## P2 级 - 次要补充

| 序号 | 核心价值 | 干系人 | 优先级 |
|------|---------|--------|--------|
| Y | 为 [干系人]，实现 [价值]，通过 [方式] | [干系人名称] | P2 |
| ... | ... | ... | ... |
```

## 使用说明

### 输入要求

用户需提供：
- 产品/项目需求描述（必填）

### 质量原则

- **完整识别**：不遗漏任何重要干系人
- **价值清晰**：价值表述要具体、可理解，避免空泛词汇
- **方式明确**：方式要与价值匹配，体现产品特色
- **优先级准确**：排序要基于干系人重要性和项目目标，不能随意

### 常见问题

**Q1：如果一个干系人有多个核心价值怎么办？**
A：分别列出，每个价值独立表述，最后在排序时综合考虑

**Q2：如果不同干系人的价值有冲突怎么办？**
A：如实列出，在优先级排序时体现出权衡（如用户体验 vs 商业收益）

**Q3：「实现什么价值」写得太泛怎么办？**
A：追问自己：这个价值能否具体化？能否量化？用户/干系人会怎么描述这个好处？

### 进阶用法

**场景 1：验证核心价值**
可以将产出的核心价值拿给实际干系人验证：「这是你最看重的价值吗？」

**场景 2：指导产品设计**
核心价值清单可以直接转化为产品 OKR 或功能优先级

**场景 3：优化商业模式**
通过分析不同干系人的价值，可以找到商业模式的切入点
