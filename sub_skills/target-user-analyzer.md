# 目标用户分析框架（Target User Analyzer）

这是一套从需求背景到完整用户画像的系统性分析方法。核心逻辑是：**基于原始需求，细分出具有针对性的用户群体，通过专业调研和数据支持，提炼出每类用户的典型特征，最终产出可指导设计的用户画像**。

分析分为 5 个步骤，针对每个用户群体输出完整的用户画像（名称、描述、特征、市场、情景）。

---

## 输入要求

用户需提供：
- **项目/产品需求描述**（必填）：说明要做什么产品或项目
- **初步的目标用户定位**（可选）：如果已有初步想法，可以提供

---

## Step 0：目标用户分类

基于原始需求，判断目标用户群体是否可以细分。

**分类原则：**
- 不同用户群体对产品/项目有**不同的影响或要求**
- 不同用户群体有**不同的侧重方向**（如使用频率、使用场景、核心诉求不同）
- 细分后的群体要有**明确的针对性**，而非简单的人口统计学划分

**输出：**

```
目标用户分类：

用户群体 A：[群体名称]
  划分依据：[为什么要单独划分这个群体]
  
用户群体 B：[群体名称]
  划分依据：...

用户群体 C：[群体名称]（如有）
  划分依据：...

（通常划分 2-4 个群体，不宜过多）
```

---

## 针对每个用户群体，完成以下分析：

---

## Step 1：目标用户描述

### 1.1 一句话定位

用一句话明确定位目标用户是谁。

**要求：**
- 清晰、简洁、准确
- 包含关键特征（年龄/职业/场景等）
- 示例：「25-35 岁在一线城市工作的职场新人，追求效率和自我提升」

**输出：**

```
目标用户定位：[一句话描述]
```

---

### 1.2 专业调研与数据支持

参考专业网站的用户分析报告、专业书籍，用规范权威的用语和数据进行定义。

**调研来源（通过 web_search 工具获取）：**
- 专业市场研究报告（如艾瑞咨询、QuestMobile、易观分析）
- 行业白皮书和权威报告
- 学术研究和专业书籍
- 竞品的用户研究报告

**调研重点：**
- 该类用户的规模和占比
- 该类用户的典型特征（权威来源的描述）
- 该类用户的行为数据和趋势

**输出：**

```
专业调研数据：

数据来源 1：[报告/书籍名称]
  关键发现：[与目标用户相关的数据和洞察]
  
数据来源 2：[报告/书籍名称]
  关键发现：...

（至少引用 2-3 个专业来源）
```

---

### 1.3 实际调查数据参考

参考实际调查数据，如用户访谈、生活追踪、问卷调查、竞品分析等。

**数据类型：**
- 用户访谈（定性数据）
- 生活追踪/观察研究
- 问卷调查（量化数据）
- 竞品分析
- 相关人员调研

**注意：**
如果用户未提供实际调查数据，AI 可以：
1. 通过 web_search 查找已有的公开调研数据
2. 基于专业调研推断合理的用户行为
3. 在输出中明确标注「基于推断，建议进行实际调研验证」

**输出：**

```
实际调查数据参考：

数据类型：[用户访谈/问卷调查/竞品分析]
  数据来源：[来源说明]
  关键发现：[与目标用户相关的洞察]

（如无实际数据，标注「建议进行以下调研以验证假设：...」）
```

---

### 1.4 用户画像补充描述

对目标用户标签的属性进行拆分和整合，准确概括出典型特征。

**描述维度：**
- 基本属性：年龄、性别、地域、职业、收入等
- 生活状态：婚姻状况、家庭结构、居住环境等
- 教育背景：学历、专业、知识结构等
- 价值观与态度：生活理念、消费观、审美倾向等

**输出：**

```
用户画像补充描述：

基本属性：
  - 年龄：[具体范围或中位数]
  - 性别：[比例分布]
  - 地域：[主要分布地区]
  - 职业：[典型职业类型]
  - 收入：[收入水平范围]

生活状态：
  - [描述典型的生活状态]

教育背景：
  - [学历水平和知识结构]

价值观与态度：
  - [核心价值观和生活态度]
```

---

## Step 2：特征属性分析

拆分出目标用户的特征属性，每个属性包含三个字段。

**特征属性格式：**

```
属性名称：[一个词概括，如「效率优先」「社交焦虑」「视觉敏感」]
  属性说明：[这个属性是什么意思]
  属性描述与影响：[这个属性如何影响用户对产品的使用或期待]
  设计启发：[基于这个属性，设计上可以怎么做]
```

---

### 2.1 行为习惯特征

思考群体的普遍行为、动作特点或日常生活习惯，提取特征属性。

**思考角度：**
- 该群体的日常作息和时间分配
- 该群体的使用习惯（手机/电脑/其他设备）
- 该群体的社交行为和互动方式
- 该群体的消费习惯和决策方式
- 该群体的学习/工作方式

**输出：**

```
行为习惯特征：

特征 1：[属性名称]
  属性说明：...
  属性描述与影响：...
  设计启发：...

特征 2：[属性名称]
  ...

（提炼所有有价值的行为习惯特征，数量不限）
```

---

### 2.2 心理特征

思考用户可能给产品带来的普遍痛点、快乐、期待，或在目前环境中会遇到的问题。

**思考角度：**
- 该群体的核心痛点是什么？
- 该群体的情绪状态和心理需求是什么？
- 该群体的期待和向往是什么？
- 该群体的恐惧和顾虑是什么？
- 该群体的自我认知和身份认同是什么？

**输出：**

```
心理特征：

特征 1：[属性名称]
  属性说明：...
  属性描述与影响：...
  设计启发：...

特征 2：[属性名称]
  ...

（提炼所有有价值的心理特征，数量不限）
```

---

### 2.3 筛选与排序

明确原始需求的设计范围，筛选出对设计有价值的特征属性，并按重要度、影响力和发生概率排序。

**排序标准：**
- **重要度**：这个特征对设计决策的影响有多大？
- **影响力**：这个特征会影响多少用户？
- **发生概率**：这个特征出现的频率有多高？

**输出：**

```
特征属性优先级排序：

【高优先级】（重要度高 + 影响力大 + 发生概率高）
1. [属性名称]
2. [属性名称]
...

【中优先级】（部分维度较高）
1. [属性名称]
...

【低优先级】（仅作参考）
1. [属性名称]
...
```

---

## Step 3：目标用户市场描述

将目标用户放大到所属的类别，描述这大概是什么样的市场，以及该市场可以覆盖的范围。

**描述维度：**
- 市场类别：这类用户属于什么市场（如「在线教育市场」「知识付费市场」）
- 市场规模：这个市场有多大（用户数量、市场规模、增长趋势）
- 市场特征：这个市场的典型特征和发展趋势
- 竞争格局：这个市场的主要玩家和竞争态势
- 覆盖范围：该用户群体在市场中的占比和分布

**数据来源：**
通过 web_search 获取市场研究报告、行业分析数据

**输出：**

```
目标用户市场描述：

市场类别：[市场名称]

市场规模：
  - 用户数量：[具体数据]
  - 市场规模：[金额/GMV 等]
  - 增长趋势：[年增长率/趋势判断]

市场特征：
  - [该市场的典型特征 1]
  - [该市场的典型特征 2]
  ...

竞争格局：
  - 主要玩家：[列举 3-5 个主要竞品]
  - 竞争态势：[市场集中度、竞争激烈程度]

覆盖范围：
  - 该用户群体在市场中的占比：[百分比或数量]
  - 地域分布：[主要分布地区]

数据来源：[引用的市场报告名称和链接]
```

---

## Step 4：典型情景编写

针对每一类目标用户，编写一个结构化的典型情景。

**情景要素：**
- 人物：具体的用户角色（虚构但典型）
- 场景：什么情况下
- 触发：为什么使用产品
- 过程：如何使用产品
- 结果：获得了什么体验或满足了什么需求

**情景结构：**

```
典型情景：[情景标题]

人物：
  - 姓名：[虚构名字]
  - 背景：[简要背景，符合用户画像]

场景：
  - 时间：[具体时间或时间段]
  - 地点：[具体场所或环境]
  - 情境：[当时发生了什么]

触发：
  - [为什么会想到使用这个产品/服务]

使用过程：
  1. [用户的第一步操作]
  2. [用户的第二步操作]
  3. [用户的后续操作]
  ...

结果：
  - 功能层面：[产品帮助用户完成了什么任务]
  - 体验层面：[用户的主观感受]
  - 价值层面：[用户获得了什么核心价值]

关键洞察：
  - [从这个情景中可以提炼出的设计洞察]
```

**情景要求：**
- 真实可信（符合用户画像和实际场景）
- 具体细致（有细节描述，不是空泛的流程）
- 有启发性（能够指导产品设计）

## 使用说明

### 输入要求

用户需提供：
- 项目/产品需求描述
- 初步的目标用户定位（可选）

### 质量原则

- **数据驱动**：所有用户描述和特征都要有数据或专业来源支持，不能凭空臆想
- **具体可用**：特征属性的「设计启发」要具体到可以直接指导设计
- **真实可信**：典型情景要符合实际，不能脱离现实
- **优先级明确**：特征属性要按重要度排序，避免面面俱到但没有重点

### 调研工具使用

本 Skill 会自动调用 web_search 工具获取：
- 市场研究报告
- 用户行为数据
- 竞品分析报告
- 行业趋势分析

如果用户有自己的调研数据，可以在输入时提供，AI 会优先使用用户提供的数据。

### 进阶用法

**场景 1：深化某个用户群体**
如果只想分析某一个特定用户群体，可以明确：「只分析 XX 群体」

**场景 2：调整细分粒度**
如果觉得用户群体划分太粗或太细，可以要求：「重新划分，粒度更细/更粗」

**场景 3：补充调研数据**
如果用户有新的调研数据，可以追加：「基于这份用户访谈，更新用户画像」
