# Requirement Architect · 需求澄清与设计引擎

> 从一句话模糊需求，到可以直接交付执行团队的完整设计案。

---

## 这是什么

当你对 AI 说"帮我做一个 X"，AI 通常会直接给方案。结果要么太通用，要么完全跑偏。

**Requirement Architect** 是一个介于你和执行团队（设计师、工程师、PM 或下游 AI）之间的**需求架构师 Skill**。它用两步完成需求到执行的全链路转化：

1. **战略澄清**（L0-L4）：把模糊需求拉到"为谁、为什么、做成什么样"的清晰共识
2. **执行设计**（L5）：把战略共识拆解为"功能模块、信息架构、用户流程、版本范围、HTML 原型"的可执行设计案

最终产物可以**直接交给**设计师、工程师、产品经理或决策层使用。

---

## 核心设计

### 方法论必须被执行，不能变成"可选项"

每个创造性需求进来，方法论必须跑一遍，不能跳过。

### L1 战略层：4 个子 Skill，缺一不可

```
L1 战略层
├── target-user-analyzer    · 目标用户分析
├── core-value-structure    · 核心价值结构化
├── scenario-analyzer       · 使用场景分析
└── ideal-state-analyzer    · 理想化状态分析（北极星）
```

### 战略之后必须有执行设计

L5 执行设计层把抽象的战略转化为可执行的设计案：

```
L5 执行设计层
├── Step 1 · 功能模块清单
├── Step 2 · 信息架构
├── Step 3 · 核心用户流程
├── Step 4 · 版本范围界定（V1 / V1.5 / V2）
└── Step 5 · HTML 原型（可选，三档）
          ├── L5-1 结构原型（Low-fi）
          ├── L5-2 视觉样品（Mid-fi）
          └── L5-3 可用原型（Hi-fi）⭐ 标准/深度档默认
```

### 深度档位：骨架不变，深度可调

| 档位 | 适用场景 | 输出深度 |
|------|---------|---------|
| 轻量档 | 个人兴趣 / 练手 / 小型内容 | 每个子 Skill 出核心结论，原型可选 L5-1 或不做 |
| 标准档 | 严肃创作 / 品牌 / 专业项目 | 完整主干，原型默认 **L5-3 可用原型** |
| 深度档 | 商业项目 / 融资 / 大型项目 | 完整执行，原型默认 **L5-3 可用原型**（完整度更高） |

## 完整架构

```
用户说："我想做一个 X"
         ↓
   L0   场景识别 + 档位判断
         ↓
   L0.5 关键前置信息对齐（避免后续大规模返工）⭐新增
         ↓
   L1   战略层（4 个子 Skill 必经，深度可调）
         ↓
   L2   定位层（客观推演 + 主观推演分层）
         ↓
   L3   形态层（场景插件）
         ↓
   L4   规格层（全自动）
         ↓
   《战略定稿》← 方向共识文档
         ↓
   L5   执行设计层（功能 / 架构 / 流程 / 版本）⭐新增
         ↓
   《产品设计案》+ HTML 原型 ⭐新增
         ↓
   交付设计师 / 工程师 / PM / 决策者
```

---

## 两种交互机制分层

- **L1 战略层**：子 Skill 引导用户深度参与，方法论驱动
- **L2-L4 执行层**：AI 主动推演，用户"默认通过 + 重点确认"；L2 的主观决策给用户真正的选择权
- **L5 执行设计层**：AI 综合推演 + 用户确认关键决策点

## 4 个子 Skill 详解

### 1. `target-user-analyzer` · 目标用户分析

从需求出发，细分用户群体，产出完整用户画像。

- **Step 0**：划分用户群体（通常 2-4 个）
- **Step 1**：一句话定位 + 专业调研数据 + 实际调查参考 + 画像描述
- **Step 2**：行为习惯特征 + 心理特征 + 优先级排序
- **Step 3**：目标用户市场规模与竞争格局
- **Step 4**：典型情景编写（结构化故事）

### 2. `core-value-structure` · 核心价值结构化

识别所有干系人，用标准句式表达每个人从项目中获得的核心价值。

- 标准句式：**为谁（干系人）+ 实现什么价值 + 通过什么方式**
- 6 种价值类型：效率提升 / 体验改善 / 能力增强 / 问题解决 / 情感满足 / 商业收益
- 按 P0 / P1 / P2 排序，明确哪些价值是项目存在的根本理由

### 3. `scenario-analyzer` · 使用场景分析

从 7 个维度横向发散（时间 / 空间 / 情绪 / 社交 / 任务 / 设备 / 生命周期），识别核心使用场景，深度画像每个场景。

- 每个场景包含：时空 / 触发 / 动作流 / 情绪曲线 / 障碍 / 期待
- 分析场景间的流转关系（承接 / 替代 / 演化）
- 识别"关键时刻"（Aha Moment / 信任时刻 / 分享时刻 / 付费时刻）

### 4. `ideal-state-analyzer` · 理想化状态分析

20 个维度系统发散项目的"北极星"——不是现实能做到什么，而是理想中做成了是什么样。

涵盖：行业标准 / 竞品启发 / AI 能力 / 游戏化 / 流行文化 / 降低用户门槛 / 仪式感 / 成长感 / 教育理念 / 平台适配……

---

## L3 场景插件

根据项目类型，自动加载对应的形态层插件：

| 插件 | 适用类型 |
|------|---------|
| `visual.md` | 视觉作品（画册、海报、插画、品牌视觉） |
| `digital.md` | 数字产品（网页、小程序、APP、工具） |
| `content.md` | 内容产品（课程、文章、视频、IP） |
| `planning.md` | 策划方案（活动、品牌、商业、营销） |
| `generic.md` | 通用兜底（其他类型） |

## 最终产出

### 《战略定稿》
- 方向共识文档，回答"为什么做、为谁做、做成什么样"
- 面向决策者、跨职能伙伴

### 《产品设计案》⭐V1.1 新增
- 执行交付文档，回答"具体要做哪些功能、长什么样、怎么排期"
- 面向设计师、工程师、产品经理

### HTML 原型（可选）⭐V1.1 新增
- 三档可选（结构原型 / 视觉样品 / 可用原型）
- **核心定位：产品原型，不是产品展示**——打开即是产品首页，不是讲解页
- 让抽象的设计案变成"打开就能用的东西"

## 安装与使用

### 安装

把整个 `requirement-architect` 文件夹放到你的 Skill 目录里。

**核心功能完全自包含**，不需要任何额外依赖。

**唯一的外部依赖**：L5 Step 5 的 L5-2 / L5-3 原型档位会调用 `frontend-design` skill 以保证视觉品质。如果环境中没有这个 skill，L5-1 线框原型不受影响；L5-2 / L5-3 会降级为基础视觉产出，仍然可用但品质有限。

### 触发条件

当用户说以下类型的话时，Skill 自动触发：

- "我想做一个 X"
- "帮我做一个 X"
- "我要做一个 X，[带一点补充]"
- "我需要一个 X"

其中 X 是需要创造的"东西"：网页、画册、课程、文章、海报、品牌、产品、活动、方案、IP、内容……

### 使用流程

1. 用户提出模糊需求
2. Skill 识别场景类型 + 判断档位
3. **L0.5** 对齐关键前置信息（避免返工）
4. **L1 战略层**：依次过 4 个子 Skill
5. **L2-L4**：AI 推演，完成战略收敛
6. 产出《战略定稿》
7. 用户确认继续 → **L5 执行设计**：功能 / 架构 / 流程 / 版本
8. 用户可选 HTML 原型档位
9. 产出《产品设计案》+ 原型
10. 交付下游团队

### 不触发的情况

- 需求已非常具体（有明确参数、案例参照、清晰方向）→ 直接做
- 纯知识性 / 技术性问题 → 不是创造需求
- 用户明确说"不用想那么多，直接给"→ 尊重用户

## 版本路线

- **V1.0**：Skill 形态，战略澄清（L0-L4），输出《需求简报》
- **V1.1（当前）**：新增 L0.5 前置对齐 + L5 执行设计层，输出《战略定稿》+《产品设计案》+ HTML 原型
- **V2（计划）**：MCP Server 形态，加入 L2.5 跃迁层（跨领域启发、假设质疑、升维版本），支持跨会话记忆、案例库积累、行业知识沉淀

---

## 二次开发指南

- **新增场景类型**：在 `L3_scene_plugins/` 下新增插件文件，不影响其他部分
- **定制子 Skill**：修改 `sub_skills/` 中的方法论，建议保留 4 个核心维度的骨架
- **调整工作流**：修改主 `SKILL.md`，建议保留"L1 必经 + 深度档位 + L5 执行设计"核心设计
