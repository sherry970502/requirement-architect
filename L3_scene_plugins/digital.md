# L3 场景插件 · 数字产品类

适用:网页、小程序、APP、工具、仪表盘、Web 应用、Landing Page 等。

## 形态维度清单

### 1. 页面/功能规模
- 单页(Landing Page / 展示页)
- 多页(3-10 页,内容型网站)
- 应用级(10+ 页,有复杂交互)

**推荐规则:**
- 核心价值偏品牌展示 → 单页 或 多页
- 核心价值偏功能服务 → 应用级
- 目标用户来一次就走 → 单页
- 目标用户反复使用 → 多页或应用级

### 2. 设备优先级
- 移动端优先(Mobile First)
- 桌面端优先(Desktop First)
- 完全响应式(同等优先)

**推荐规则:**
- 目标用户是个人消费者 → 移动端优先
- 目标用户是专业用户/B 端 → 桌面端优先
- 不确定 → 完全响应式

### 3. 交互复杂度
- 静态展示(只读、无需交互)
- 轻交互(表单、筛选、简单反馈)
- 重交互(账号、数据、实时)

**推荐规则:**
- 内容型(博客、作品集、品牌站) → 静态展示 或 轻交互
- 工具型 → 轻交互 或 重交互
- SaaS/平台型 → 重交互

### 4. 数据/后端需求
- 纯前端静态(无后端)
- 轻量后端(表单收集、简单数据)
- 完整后端(账号、数据库、API)

**推荐规则:** 基于交互复杂度自然推出。展示类项目建议纯前端(Vercel/Netlify 静态托管),成本和维护最优。

### 5. 技术栈倾向
- 极简(HTML/CSS/JS 单文件)
- 现代前端框架(React/Vue + Tailwind)
- 全栈框架(Next.js / Nuxt 等)

**推荐规则:** 默认给"现代前端框架"建议。除非是极简展示页(单文件更合适)或有明确后端需求(全栈框架)。

## 需要高亮让用户关注的维度

- **页面/功能规模**(决定项目量级和时间周期,差距巨大)
- **设备优先级**(决定设计思路,不能后期调整)

## L4 规格层自动推导规则

### 视觉规格
- **设计尺寸**:
  - 桌面端主稿 1440×900 或 1920×1080
  - 移动端主稿 375×812 或 414×896
- **栅格系统**:12 列栅格(桌面)/ 4-6 列(移动)
- **配色**:主色 + 1-2 辅色 + 中性色体系

### 性能与兼容
- 首屏加载 < 3 秒
- 兼容:Chrome/Safari/Firefox/Edge 最新版 + iOS Safari / Android Chrome

### 交付形式
- 源码 + 部署说明
- 设计稿(Figma 或等同)
- 内容清单(文案、图片、资源)

### 部署建议
- 纯前端静态 → Vercel / Netlify / Cloudflare Pages
- 有后端 → 根据技术栈决定(Vercel 全栈 / 云服务器)
- 国内访问 → 考虑国内部署方案(阿里云/腾讯云 + 备案)
