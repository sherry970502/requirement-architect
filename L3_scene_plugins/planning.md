# L3 场景插件 · 策划方案类

适用:活动策划、品牌策划、营销方案、商业方案、产品方案、项目规划等。

## 形态维度清单

### 1. 方案深度
- 创意概念(只到创意和方向)
- 执行方案(有具体的操作步骤)
- 完整可落地(含预算、排期、责任人、KPI)

**推荐规则:**
- 用于内部讨论/脑暴 → 创意概念
- 用于跨团队协同 → 执行方案
- 用于汇报/立项/融资 → 完整可落地

### 2. 方案用途
- 内部决策依据
- 对外提案(给客户/合作方)
- 汇报材料(给上级/投资人)
- 执行指南(给执行团队)

**推荐规则:** 直接影响格式和重点——对外提案要有视觉化的 Slide 感,执行指南要有清单和流程,汇报材料要有数据和逻辑链。

### 3. 呈现形态
- 文档(Word 长文)
- Slide(PPT/Keynote)
- 一页纸(One-pager)
- 组合(多形态同步输出)

**推荐规则:**
- 内部决策 → 文档 或 Slide
- 对外提案 → Slide 或 组合
- 汇报 → Slide
- 执行指南 → 文档

### 4. 规模/量级
- 小型(单次活动、小型项目、试水方案)
- 中型(季度项目、区域规划、中等预算)
- 大型(年度战略、全国铺开、大预算)

**推荐规则:** 基于用户在 L1 阶段透露的信息(预算量级、团队规模、时间跨度)推出。如果完全没信息,默认"中型",并在简报里标注"假设为中型规模,如需调整请告知"。

### 5. 风险/假设管理
- 列出关键假设,标注风险(严肃项目)
- 仅给主方案,不单独列风险(轻量项目)

**推荐规则:** 涉及融资、重大决策、大预算的项目,必须列假设和风险。其他情况可省略。

---

## 需要高亮让用户关注的维度

- **方案深度**(决定工作量,相差 3-5 倍)
- **方案用途**(决定写作风格,不能通用)
- **规模/量级**(决定方案的数字和野心,猜错会很尴尬)

## L4 规格层自动推导规则

### 文档类
- **字数**:
  - 创意概念:1500-3000 字
  - 执行方案:3000-8000 字
  - 完整方案:8000-20000 字
- **结构**:
  - 背景 → 目标 → 策略 → 执行 → 预算/排期 → 评估
  - 每章节有小标题、关键数据、必要时的图表

### Slide 类
- **张数**:
  - 创意概念:10-20 页
  - 执行方案:20-40 页
  - 完整方案:40-80 页
- **结构**:
  - 封面 + 议程 + 背景 + 洞察 + 创意/策略 + 执行 + 预算排期 + 团队 + 总结
- **设计标准**:每页一个核心论点,视觉化优先

### 一页纸(One-pager)
- **字数**:300-800 字
- **结构**:Hook(一句话)+ Why(为什么做)+ What(做什么)+ How(怎么做)+ Why-us(为什么是我们/你)
- **用途**:融资、BD、内部 pitch

### 通用要求
- **数据支撑**:关键判断要有数据或调研依据,标明来源
- **可执行性**:涉及执行的部分要可量化、可验收
- **视觉化**:重点逻辑/数据要用图表或架构图,不要全是文字

## 特别说明

策划方案类项目,**强烈建议在 L1 阶段引导用户调用两个深度 skill**:

- `target-user-analyzer`:方案必须有扎实的目标用户分析
- `core-value-structure`:方案必须说清楚为谁创造什么价值

没有这两个地基的策划方案,再精美也会被一眼看穿是"浮在上面的"。在主 skill 的 L1 阶段,如果识别到是策划方案类项目,应主动建议:

> "策划方案类的项目,我强烈建议我们先用两个深度工具把地基打好:
> 1. `target-user-analyzer` - 完成目标用户的深度画像
> 2. `core-value-structure` - 完成核心价值的结构化
>
> 这两步做完之后,后面的策略、执行、预算都会有清晰的锚点。需要我调用吗?"
